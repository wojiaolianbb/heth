# Heth 项目技术设计方案

## 一、功能路由表

### 1. 饮水记录 `/water`

#### 功能点
- ✅ 显示今日饮水进度
- ✅ 快捷加水按钮（100/250/500ml）
- ✅ 自定义毫升数输入
- ✅ 今日记录列表
- ✅ 删除记录

#### API
- ✅ `GET /api/water?date=YYYY-MM-DD` - 查询指定日期饮水记录
- ✅ `POST /api/water` - 添加饮水记录
- ✅ `DELETE /api/water/[id]?date=YYYY-MM-DD` - 删除记录

#### 数据模型
```typescript
type WaterLog = {
  id: string;
  date: string;
  time: string;
  amountMl: number;
  createdAt: string;
}
```

---

### 2. 体重记录 `/body`

#### 功能点
- ✅ 显示最新体重和BMI
- ✅ 输入体重（支持小数）
- ✅ 自动计算BMI
- ✅ 历史记录列表
- ❌ 7天/30天趋势图表

#### API
- ✅ `GET /api/body` - 查询所有体重记录
- ✅ `POST /api/body` - 添加体重记录
- ❌ `DELETE /api/body/[id]` - 删除记录
- ❌ `GET /api/body/trend?days=7` - 趋势数据

#### 数据模型
```typescript
type BodyMetric = {
  id: string;
  date: string;
  weightKg: number;
  bmi: number;
  note?: string;
  createdAt: string;
}
```

---

### 3. 睡眠记录 `/sleep`

#### 功能点
- ✅ 入睡时间输入（HH:mm）
- ✅ 起床时间输入（HH:mm）
- ✅ 自动计算时长（支持跨天）
- ✅ 睡眠质量选择（好/一般/差）
- ✅ 今日记录显示
- ❌ 7天睡眠节律图表
- ❌ 平均睡眠时长统计

#### API
- ✅ `GET /api/sleep?date=YYYY-MM-DD` - 查询指定日期睡眠记录
- ✅ `POST /api/sleep` - 添加睡眠记录
- ❌ `DELETE /api/sleep/[id]` - 删除记录
- ❌ `GET /api/sleep/stats?days=7` - 睡眠统计

#### 数据模型
```typescript
type SleepLog = {
  id: string;
  date: string;
  bedTime: string;
  wakeTime: string;
  durationMinutes: number;
  quality: 'good' | 'ok' | 'poor';
  note?: string;
  createdAt: string;
}
```

---

### 4. 运动记录 `/exercise`

#### 功能点
- ❌ 选择运动类型（步行/跑步/骑行/力量/拉伸/球类/其他）
- ❌ 输入运动时长（分钟）
- ❌ 选择运动强度（低/中/高）
- ❌ 可选：输入步数
- ❌ 今日记录列表
- ❌ 删除记录
- ❌ 周运动时长统计

#### API
- ❌ `GET /api/exercise?date=YYYY-MM-DD` - 查询指定日期运动记录
- ❌ `POST /api/exercise` - 添加运动记录
- ❌ `DELETE /api/exercise/[id]` - 删除记录
- ❌ `GET /api/exercise/stats?type=week` - 周统计

#### 数据模型
```typescript
type ExerciseLog = {
  id: string;
  date: string;
  type: 'walk' | 'run' | 'cycle' | 'strength' | 'stretch' | 'sport' | 'other';
  durationMinutes: number;
  intensity: 'low' | 'medium' | 'high';
  steps?: number;
  note?: string;
  createdAt: string;
}
```

---

### 5. 饮食记录 `/meals`

#### 功能点
- ❌ 选择餐次（早餐/午餐/晚餐/加餐）
- ❌ 输入食物名称
- ❌ 可选：输入营养数据（热量/蛋白质/碳水/脂肪）
- ❌ 今日餐食列表
- ❌ 删除记录
- ❌ 今日营养汇总

#### API
- ❌ `GET /api/meals?date=YYYY-MM-DD` - 查询指定日期饮食记录
- ❌ `POST /api/meals` - 添加饮食记录
- ❌ `DELETE /api/meals/[id]` - 删除记录
- ❌ `GET /api/meals/summary?date=YYYY-MM-DD` - 营养汇总

#### 数据模型
```typescript
type MealLog = {
  id: string;
  date: string;
  time: string;
  mealType: 'breakfast' | 'lunch' | 'dinner' | 'snack';
  foodName: string;
  caloriesKcal?: number;
  proteinG?: number;
  carbsG?: number;
  fatG?: number;
  note?: string;
  createdAt: string;
}
```

---

### 6. Dashboard `/dashboard`

#### 功能点
- ✅ 今日饮水卡片（显示总量/目标）
- ✅ 最新体重卡片（显示体重/BMI）
- ✅ 昨晚睡眠卡片（显示时长/质量）
- ❌ 今日运动卡片（显示总时长/类型）
- ❌ 今日饮食卡片（显示餐数/总热量）
- ❌ 快速记录按钮

#### API
- ✅ 调用现有API组合数据
- ❌ `GET /api/dashboard?date=YYYY-MM-DD` - 一次性获取所有今日数据

---

### 7. 用户资料 `/profile`

#### 功能点
- ❌ 编辑姓名
- ❌ 设置身高（用于BMI计算）
- ❌ 设置饮水目标
- ❌ 设置目标体重
- ❌ 设置每日热量目标

#### API
- ❌ `GET /api/profile` - 获取用户资料
- ❌ `PUT /api/profile` - 更新用户资料

#### 数据模型
```typescript
type UserProfile = {
  name?: string;
  heightCm: number;
  targetWeightKg?: number;
  dailyWaterGoalMl: number;
  dailyCalorieGoal?: number;
  createdAt: string;
  updatedAt: string;
}
```

---

### 8. 复盘分析 `/review`

#### 功能点
- ❌ 选择时间范围（7天/30天）
- ❌ 饮水趋势图
- ❌ 体重趋势图
- ❌ 睡眠时长趋势图
- ❌ 运动时长统计
- ❌ 数据缺口提示
- ❌ 周报/月报生成

#### API
- ❌ `GET /api/stats?type=water&days=7` - 饮水统计
- ❌ `GET /api/stats?type=body&days=30` - 体重统计
- ❌ `GET /api/stats?type=sleep&days=7` - 睡眠统计
- ❌ `GET /api/stats?type=exercise&days=7` - 运动统计
- ❌ `GET /api/review/report?days=7` - 生成报告

## 三、多Agent并行开发方案

### Phase 1: 基础记录功能（3个Agent并行）

#### Agent 1: 运动记录
**任务**:
- 实现 `lib/types/exercise.ts`
- 实现 `app/api/exercise/route.ts` (GET/POST)
- 实现 `app/api/exercise/[id]/route.ts` (DELETE)
- 实现 `app/exercise/page.tsx`

**输入**: 数据模型定义
**输出**: 完整的运动记录功能
**预计时间**: 2小时

---

#### Agent 2: 饮食记录
**任务**:
- 实现 `lib/types/meals.ts`
- 实现 `app/api/meals/route.ts` (GET/POST)
- 实现 `app/api/meals/[id]/route.ts` (DELETE)
- 实现 `app/api/meals/summary/route.ts`
- 实现 `app/meals/page.tsx`

**输入**: 数据模型定义
**输出**: 完整的饮食记录功能
**预计时间**: 2小时

---

#### Agent 3: 用户资料
**任务**:
- 实现 `lib/types/profile.ts`
- 实现 `app/api/profile/route.ts` (GET/PUT)
- 实现 `app/profile/page.tsx`
- 修改其他页面读取用户资料（身高、目标）

**输入**: 数据模型定义
**输出**: 完整的用户资料功能
**预计时间**: 1.5小时

---

### Phase 2: 增强现有功能（3个Agent并行）

#### Agent 4: 完善已有记录功能
**任务**:
- 为体重/睡眠/饮水添加 DELETE API
- 为所有记录添加编辑功能
- 统一错误处理

**输入**: 现有代码
**输出**: 增强的CRUD功能
**预计时间**: 1小时

---

#### Agent 5: 统计API
**任务**:
- 实现 `app/api/stats/route.ts`
- 支持查询参数：type, days, startDate, endDate
- 返回聚合数据：总量、平均值、最大/最小值

**输入**: 现有数据文件
**输出**: 统一的统计接口
**预计时间**: 1.5小时

---

#### Agent 6: Dashboard优化
**任务**:
- 实现 `app/api/dashboard/route.ts`
- 一次请求返回所有今日数据
- 更新 `app/dashboard/page.tsx` 使用新API
- 添加运动和饮食卡片

**输入**: 现有Dashboard代码
**输出**: 优化的Dashboard
**预计时间**: 1小时

---

### Phase 3: 数据可视化（2个Agent并行）

#### Agent 7: 趋势图表组件
**任务**:
- 安装 recharts
- 实现 `components/charts/LineChart.tsx`
- 实现 `components/charts/BarChart.tsx`
- 实现通用图表配置

**输入**: 数据格式定义
**输出**: 可复用的图表组件
**预计时间**: 2小时

---

#### Agent 8: 复盘分析页
**任务**:
- 实现 `app/review/page.tsx`
- 集成图表组件
- 显示各项数据趋势
- 实现时间范围切换

**输入**: 统计API + 图表组件
**输出**: 完整的复盘页面
**预计时间**: 2小时

## 四、Agent工作流脚本

```javascript
export const meta = {
  name: 'heth-parallel-development',
  description: '并行开发Heth项目剩余功能',
  phases: [
    { title: 'Phase 1', detail: '基础记录功能' },
    { title: 'Phase 2', detail: '增强现有功能' },
    { title: 'Phase 3', detail: '数据可视化' }
  ]
};

// Phase 1: 基础记录功能（并行）
phase('Phase 1');
const phase1 = await parallel([
  () => agent(`实现运动记录完整功能：
    1. 创建 lib/types/exercise.ts
    2. 创建 app/api/exercise/route.ts (GET/POST)
    3. 创建 app/api/exercise/[id]/route.ts (DELETE)
    4. 创建 app/exercise/page.tsx
    参考现有的 water 功能实现`, 
    { label: '运动记录', phase: 'Phase 1', isolation: 'worktree' }),
  
  () => agent(`实现饮食记录完整功能：
    1. 创建 lib/types/meals.ts
    2. 创建 app/api/meals/route.ts (GET/POST)
    3. 创建 app/api/meals/[id]/route.ts (DELETE)
    4. 创建 app/meals/page.tsx
    5. 支持营养汇总
    参考现有的 water 功能实现`, 
    { label: '饮食记录', phase: 'Phase 1', isolation: 'worktree' }),
  
  () => agent(`实现用户资料管理：
    1. 创建 lib/types/profile.ts
    2. 创建 app/api/profile/route.ts (GET/PUT)
    3. 创建 app/profile/page.tsx
    4. 修改 body 页面读取身高`, 
    { label: '用户资料', phase: 'Phase 1', isolation: 'worktree' })
]);

log('Phase 1 完成：基础记录功能');

// Phase 2: 增强现有功能（并行）
phase('Phase 2');
const phase2 = await parallel([
  () => agent(`为已有功能添加删除API：
    1. 实现 DELETE /api/body/[id]
    2. 实现 DELETE /api/sleep/[id]
    3. 在各页面添加删除按钮`, 
    { label: '完善CRUD', phase: 'Phase 2' }),
  
  () => agent(`实现统一统计API：
    1. 创建 app/api/stats/route.ts
    2. 支持参数：type, days, startDate, endDate
    3. 返回聚合数据`, 
    { label: '统计API', phase: 'Phase 2' }),
  
  () => agent(`优化Dashboard：
    1. 创建 app/api/dashboard/route.ts
    2. 一次请求返回所有今日数据
    3. 添加运动和饮食卡片`, 
    { label: 'Dashboard优化', phase: 'Phase 2' })
]);

log('Phase 2 完成：功能增强');

// Phase 3: 数据可视化（并行）
phase('Phase 3');
const phase3 = await parallel([
  () => agent(`创建图表组件：
    1. 安装 recharts
    2. 创建 components/charts/LineChart.tsx
    3. 创建 components/charts/BarChart.tsx`, 
    { label: '图表组件', phase: 'Phase 3' }),
  
  () => agent(`实现复盘分析页：
    1. 创建 app/review/page.tsx
    2. 显示各项趋势图
    3. 支持时间范围切换`, 
    { label: '复盘页面', phase: 'Phase 3' })
]);

log('Phase 3 完成：数据可视化');

return {
  success: true,
  summary: '所有功能开发完成',
  phases: [
    { name: 'Phase 1', agents: phase1.filter(Boolean).length, status: 'completed' },
    { name: 'Phase 2', agents: phase2.filter(Boolean).length, status: 'completed' },
    { name: 'Phase 3', agents: phase3.filter(Boolean).length, status: 'completed' }
  ]
};
```
